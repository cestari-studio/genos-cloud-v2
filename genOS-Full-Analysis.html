Arquitetura do Sistema

Fluxo Principal: Request Pipeline

Request HTTP → CORS Middleware → Tenant Resolution (x-tenant-id header | slug | env fallback) → Identity Middleware (JWT → Wix Headers → Generic → Query params) → RBAC Check (requireRole | requirePermission | requireAuthenticated) → Route Handler → Service Layer → Supabase (RLS) / Wix API / Gemini / Claude

Fluxo: Geração de Conteúdo com Compliance

POST /api/ai/generate-and-save → agentEnvelope.buildEnvelope() [Brand DNA + System Prompt + Contexto] → aiRouter.generateContent() [Rota para Gemini/Claude/Local] → masterCompliance.checkCompliance() [Validação 4 camadas: 0-100] → supabase.insert(posts) [Salvar se score ≥ 75] → pricingResolver.calculateCost() [Tracking tokens + custo] → activityFeed.emitFeedEvent() [Log no activity_log]

Fluxo: CSV Sync Bidirecional

Chokidar detecta mudança em /projects/{tenant}/ → csvWatcher.detectChanges() [Comparar hashes] → syncEngine.syncDbToCsv() [Export DB → CSV local] OU → syncEngine.syncCsvToWix() [Push CSV → Wix CMS] → Field mapping via csv_registry [Mapeamento por coleção] → csvEngine.writeCsv() [Formato: comma (DB) | semicolon+BOM (Excel)] → activity_log + csv_sync_log [Audit trail]

Fluxo: Feedback Loop com Auto-Revisão

POST /webhooks/wix [Cliente atualiza conteúdo no Wix] → feedbackQueue.insert() → processFeedbackQueue() [async] → handleRevisionRequest() [Se needs_revision] → aiRouter.generateContent() [Regenerar com feedback] → masterCompliance.checkCompliance() → posts.update() [Substituir conteúdo] → wixClient.updateItem() [Sincronizar com Wix] → sentimentPulse.analyzeSentiment() [Detectar padrões]

⚠️ CORS Vulnerability

Todas as 3 Edge Functions usam:

'Access-Control-Allow-Origin': '*'

Isso permite que QUALQUER site faça requisições para seus endpoints. Deveria ser restrito a:

'Access-Control-Allow-Origin': 'https://genos-cloud-v2.vercel.app'

⚠️ Auto-Assign super_admin

O wix-auth-bridge atribui super_admin para TODOS os novos tenants auto-provisionados. Qualquer pessoa que fizer login recebe acesso total.

Fix: Atribuir client_user por padrão e promover manualmente.

Fluxo do wix-auth-bridge

POST /wix-auth-bridge { email, password } 1. Busca tenant por email no tenant_members 2. Se não existe: a. Cria novo tenant (name = email prefix, slug = email prefix) b. Cria auth user via admin.createUser() c. Cria tenant_member com role = 'super_admin' ← BUG d. Define parent_tenant_id = master tenant 3. Se existe: a. Tenta sign in com email/password b. Se falha: admin.updateUser() reset senha ← PERIGOSO 4. Retorna { user, session, tenant }

Análise de Segurança

🚨 P0-1: GEMINI_API_KEY Exposta no .env

Arquivo: .env (na raiz do projeto)

Risco: A chave da API Gemini está commitada no repositório. Qualquer pessoa com acesso ao repo pode usar os créditos ou fazer chamadas API em seu nome.

Ação: Rotacionar a chave IMEDIATAMENTE no Google Cloud Console, depois limpar do histórico git.

🚨 P0-2: CORS = '*' nas Edge Functions

Arquivos: Todas as 3 Edge Functions

Risco: Qualquer site pode fazer requisições para seus endpoints, incluindo ataques CSRF.

Ação: Restringir para domínios conhecidos: genos-cloud-v2.vercel.app e localhost:5173.

🚨 P0-3: Auto-Assign super_admin

Arquivo: supabase/functions/wix-auth-bridge/index.ts

Risco: Todos os novos usuários recebem role super_admin, tendo acesso total à plataforma.

Ação: Mudar default para client_user.

🚨 P0-4: Reset de Senha Silencioso

Arquivo: wix-auth-bridge

Risco: Se o login falha, a function reseta a senha silenciosamente via admin.updateUser(). Isso é um vetor de takeover de conta.

Ação: Remover o reset automático. Usar fluxo de recuperação explícito.

// ANTES (inseguro):

const ALLOWED_ORIGINS = [
  'https://genos-cloud-v2.vercel.app',
  'http://localhost:5173',

const globalLimiter = rateLimit({
