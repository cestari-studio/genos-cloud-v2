ACL Modelo de Permissões por Recurso

O genOS opera com 3 níveis de tenant (depth_level): Master (0), Agency (1) e Client (2+). Cada recurso tem permissões granulares por nível.

Master (0)
  • Ativar/desativar Schedule para Agencies
  • Definir preço e limites por plano
  • Visualizar todos os schedules
  • Configurar feature flags globais
  • Override de limites de posts

Agency (1)

  • Configurar slots de horário
  • Ajustar prioridade de fila
  • Deletar agendamentos
  • Ver métricas de publicação
  • Aprovar/rejeitar posts agendados

DB Quality Gate — P1: Database Schema

PROMPT 5.1 — QUALITY GATE DATABASE
Atue como engenheiro PostgreSQL sênior do genOS Cloud Platform v2. CONTEXTO: - O Quality Gate é a "Fila de Governança" do Content Factory - Valida posts contra regras de constraint definidas no Brand DNA - Constraint types: character_count, hashtag_count, tone_adherence, frame_limit, forbidden_words - Cada post gerado pela IA passa pelo Quality Gate antes de poder ser approved - Já existe no BrandDna.tsx: character limits por formato, forbidden words, hashtag rules, tone settings TAREFA — Migration SQL: 1. CREATE TABLE constraint_rules: - id UUID PRIMARY KEY - tenant_id UUID NOT NULL REFERENCES tenants(id) - rule_type TEXT NOT NULL CHECK (rule_type IN ('character_count','hashtag_count','tone_adherence','frame_limit','forbidden_words','cta_required','language_check','brand_voice_score')) - format_target TEXT (feed, carrossel, stories, reels, all) - config JSONB NOT NULL (ex: {"min": 100, "max": 280} ou {"threshold": 0.85} ou {"words": ["promoção","grátis"]}) - severity TEXT DEFAULT 'violation' CHECK (severity IN ('violation','warning','info')) - enabled BOOLEAN DEFAULT true - created_at TIMESTAMPTZ DEFAULT now() - UNIQUE(tenant_id, rule_type, format_target) 2. CREATE TABLE quality_evaluations: - id UUID PRIMARY KEY - post_id UUID NOT NULL REFERENCES posts(id) ON DELETE CASCADE - tenant_id UUID NOT NULL - overall_score NUMERIC(5,2) (0-100) - status TEXT DEFAULT 'pending' CHECK (status IN ('pending','passed','failed','exception_approved')) - evaluated_at TIMESTAMPTZ DEFAULT now() - evaluated_by TEXT DEFAULT 'system' (system | user_uuid) 3. CREATE TABLE constraint_results: - id UUID PRIMARY KEY - evaluation_id UUID NOT NULL REFERENCES quality_evaluations(id) ON DELETE CASCADE - rule_id UUID NOT NULL REFERENCES constraint_rules(id) - passed BOOLEAN NOT NULL - actual_value TEXT (ex: "387" para char count) - expected_value TEXT (ex: "max:280") - severity TEXT - detail TEXT (ex: "387 caracteres excede o limite de 280") 4. RLS POLICIES: - constraint_rules: Master CRUD all, Agency CRUD own, Client SELECT own - quality_evaluations: Master/Agency SELECT/UPDATE all under their tree, Client SELECT own posts - constraint_results: segue evaluation access 5. FUNCTION evaluate_post(p_post_id UUID) RETURNS UUID (evaluation_id): - Buscar post + tenant_id - Buscar constraint_rules ativas do tenant - Para cada regra aplicável ao content_type do post: - Executar validação (character_count → length(title+body), etc) - Inserir resultado em constraint_results - Calcular overall_score (% de regras passadas, ponderado por severity) - Inserir quality_evaluations - Se overall_score >= 80 E zero violations → status = 'passed' - Senão → status = 'failed' - Retornar evaluation_id 6. TRIGGER: after INSERT on posts → chamar evaluate_post automaticamente 7. FUNCTION seed_default_constraints(p_tenant_id UUID): - Inserir regras padrão baseadas nos character limits que já existem no Brand DNA - Tipos: feed (max 2200 chars), stories (max 250), reels título (max 100), carrossel (max 2200) - Hashtag limit: max do Brand DNA (ex: 30) - Forbidden words: lista do Brand DNA - Tone adherence: threshold 0.75 default Gere SQL completo com comentários.
