🔴 Bug Fix — Social Connect Buttons

Os botões "Conectar Instagram" e "Conectar Facebook" no Settings não funcionam. Três bugs encontrados e corrigidos.

Bug 1 — Double Destructure no handleConnect

CRITICAL — Botão não faz nada

Sintoma: Clicar em "Conectar Instagram" ou "Conectar Facebook" não abre nenhuma janela/redirect.

Causa raiz: O api.edgeFn() já faz unwrap do campo data na resposta. O componente faz um segundo destructure, resultando em undefined.

ui-react/src/components/SocialConnectionsTab.tsx

Código com bug (linhas 48-59)

- const handleConnect = async (provider: 'instagram' | 'facebook') => {
-     try {
-         const { data } = await api.edgeFn<any>('meta-oauth', {
-             action: 'get_auth_url',
-             provider,
-             tenant_id: activeTenantId
-         });
-         window.location.href = data.auth_url;  // ← data = undefined!
-     } catch (err: any) {
-         showToast('Erro ao iniciar conexão', err.message, 'error');
-     }
- };

Código corrigido

+ const handleConnect = async (provider: 'instagram' | 'facebook') => {
+     try {
+         // edgeFn already unwraps { data: ... }, so result IS the inner object
+         const result = await api.edgeFn<{ auth_url: string }>('meta-oauth', {
+             action: 'get_auth_url',
+             provider,
+             tenant_id: activeTenantId
+         });
+         if (!result?.auth_url) throw new Error('URL de autenticação não recebida');
+         window.location.href = result.auth_url;
+     } catch (err: any) {
+         showToast('Erro ao iniciar conexão', err.message, 'error');
+     }
+ };

Explicação: A função edgeFn em api.ts (linha 206) já verifica se a resposta tem data e retorna responseJson.data. A edge function meta-oauth retorna {"data": {"auth_url": "https://..."}}, então edgeFn já devolve {"auth_url": "https://..."}. Fazer const { data } = ... tenta extrair .data de algo que não tem essa propriedade → undefined.

Bug 2 — Parâmetros errados no handleDisconnect

MEDIUM — Desconectar não funciona

Sintoma: Clicar "Desconectar" retorna erro 500 da edge function.

Causa raiz: O frontend envia connectionId (camelCase), mas a edge function espera connection_id (snake_case) + tenant_id.

ui-react/src/components/SocialConnectionsTab.tsx

Código com bug (linhas 61-69)

- const handleDisconnect = async (connectionId: string) => {
-     try {
-         await api.edgeFn('meta-oauth', { action: 'disconnect', connectionId });
-         // ← falta tenant_id e nome do campo é connectionId em vez de connection_id
-     ...

Código corrigido

+ const handleDisconnect = async (connectionId: string) => {
+     try {
+         await api.edgeFn('meta-oauth', {
+             action: 'disconnect',
+             connection_id: connectionId,
+             tenant_id: activeTenantId
+         });
+         showToast('Sucesso', 'Conexão removida', 'success');
+         fetchConnections();
+     } catch (err: any) {
+         showToast('Erro ao desconectar', err.message, 'error');
+     }
+ };

Bug 3 — Falta ig_account_id no select_page

MEDIUM — Instagram não conecta via Facebook Login

Sintoma: Após autenticar com Facebook e selecionar uma página, a conta Instagram Business vinculada à página NÃO é salva.

Causa raiz: O callback não envia ig_account_id nem provider para a edge function.

ui-react/src/pages/auth/SocialCallback.tsx

Código com bug (linhas 69-76)

- await api.edgeFn('meta-oauth', {
-     action: 'select_page',
-     tenant_id: stateData.tenant_id,
-     page_id: page.id,
-     page_name: page.name,
-     page_token: page.access_token
-     // ← falta ig_account_id e provider
- });

Código corrigido

+ await api.edgeFn('meta-oauth', {
+     action: 'select_page',
+     tenant_id: stateData.tenant_id,
+     page_id: page.id,
+     page_name: page.name,
+     page_token: page.access_token,
+     ig_account_id: page.instagram_business_account?.id || null,
+     provider: stateData.provider,
+ });

Explicação: A API do Facebook retorna instagram_business_account no objeto da página (quando existe). A edge function meta-oauth precisa desse ID para buscar os dados da conta IG e salvar como conexão separada. Sem isso, só o Facebook conecta.

✅ Resumo dos Arquivos Alterados

  • SocialConnectionsTab.tsx — Bug 1 (handleConnect) + Bug 2 (handleDisconnect)
  • SocialCallback.tsx — Bug 3 (handleSelectPage)

Nenhuma alteração na edge function — o problema era 100% frontend.

Padrão a lembrar: api.edgeFn() já faz unwrap de {data: ...}. Nunca fazer const { data } = await api.edgeFn(...) — usar const result = await api.edgeFn(...).
