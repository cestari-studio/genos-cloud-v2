Detalhamento — Features Críticas

NI-001 Client Onboarding Flow Planejado em: Arquitetura v1.0

Fluxo completo: cliente navega site Cestari Studio → seleciona serviço/plano → preenche briefing → contrato gerado → acesso provisionado.

Onde foi planejado: genOS-Arquitetura-Completa.md menciona CSV collections para accounts, briefs, contracts e portal.

Estado atual: Tabelas existem no schema como estrutura de dados, mas nenhuma UI, Edge Function ou automação foi construída. A provisão de cada tenant é 100% manual no Supabase Dashboard.

Impacto: Cada novo cliente requer horas de setup manual pelo Master. Limita escalabilidade a ~2 novos clientes/semana.

NI-002 Geração Automática de Contratos Planejado em: Arquitetura v1.0

Sistema para gerar contrato PDF com dados do cliente, plano selecionado, escopo de serviços e termos da agência. Integração com assinatura digital.

Onde foi planejado: README.md lista collection contracts no CSV sync pipeline.

Estado atual: A collection existe como schema de dados mas nenhum template, gerador PDF ou UI foi implementado. Contratos são feitos em ferramentas externas (Word/PDF).

Impacto: Processo jurídico completamente desvinculado do genOS. Sem rastreabilidade de contratos por tenant.

NI-003 Formulário de Briefing / Escopo Planejado em: IBM Catalog + Arquitetura v1.0

Formulário estruturado (ou conversa guiada por IA) para coletar: objetivos, público-alvo, tom de voz, referências visuais, concorrentes e metas de conteúdo. Alimenta automaticamente o Brand DNA.

Onde foi planejado: IBM_Catalog_genOS.html (Briefing Agent no watsonx Orchestrate) e genOS-Arquitetura-Completa.md (briefs collection). A arquitetura menciona que Brand DNA pode ser "gerada pela IA a partir de um briefing puro".

Estado atual: O formulário de briefing NUNCA foi construído. O BrandDna.tsx tem bugs críticos (10 campos fantasma, mismatch PT/EN). Sem briefing estruturado, o Brand DNA é preenchido manualmente e de forma incompleta.

Impacto: Conteúdo gerado pela IA é genérico porque o Brand DNA não reflete o briefing real do cliente. 15% de eficiência perdida por fallbacks.

NI-004 Portal do Cliente (Self-Service) Planejado em: Etapa C + Arquitetura v1.0

Interface web pública onde potenciais clientes veem pacotes, comparam planos, iniciam trial e compram diretamente — sem contatar a agência.

Onde foi planejado: genOS-Arquitetura-Completa.md (CSV collection portal) e PROXIMAS_ETAPAS.md (Etapa C: Billing/Pricing/Quotas).

Estado atual: Nenhuma landing page, página de pricing ou checkout existe. Todo o processo de venda é offline.

Impacto: Limita crescimento — cada venda depende de contato direto. Escala linear com horas do fundador.
