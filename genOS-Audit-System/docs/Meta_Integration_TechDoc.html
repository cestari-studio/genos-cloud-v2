Este documento define a arquitetura técnica completa, os prompts de engenharia, e o plano de execução por fases para integrar as APIs do Meta Platform (Instagram, Facebook e WhatsApp) diretamente ao genOS Cloud Platform v2 Content Factory.

O objetivo é permitir que agências (tenants nível 1) e seus clientes (tenants nível 2) possam:

  • Publicar conteúdo automaticamente no Instagram (feed, stories, reels, carousels) e Facebook (posts, fotos, vídeos)
  • Agendar publicações com base no calendário do Content Factory
  • Gerenciar aprovações e automação via WhatsApp Business Platform
  • Armazenar tokens OAuth por tenant com segurança (Vault pattern no Supabase)
ℹ
Stack Base: React 18.3 + TypeScript 5.6 + Vite 6.0 + Supabase (PostgreSQL + Edge Functions + Realtime) + Carbon Design System v11 (g100) + Vercel

1.3 App Review Process

⚠
Desde 2025, ~90% dos risk assessments são automatizados por IA. Porém, a qualidade do screencast e a clareza da privacy policy continuam sendo os maiores motivos de rejeição. Rejections adicionam 3-5 dias por tentativa.

O App Review exige um screencast de vídeo demonstrando exatamente como cada permissão é utilizada. Cada permissão deve ser justificada individualmente.

Checklist Completo para App Review
  1. ✅ Privacy Policy publicada e acessível (HTTPS)
  2. ✅ Data Deletion Callback implementado e respondendo 200
  3. ✅ Deauthorize Callback implementado
  4. ✅ Screencast de 3-5 min mostrando:
    • Login OAuth completo (grant → redirect → token)
    • Para cada permission: exatamente onde/como é usada na UI
    • Fluxo de publicação end-to-end (criar post → preview → publish)
    • Settings onde usuário pode desconectar
  5. ✅ Business Verification completa (CNPJ + domínio)
  6. ✅ Todas as permissões justificadas com texto explicativo
  7. ✅ Test Users adicionados para o reviewer
  8. ✅ App em modo "Live" (não Development)
