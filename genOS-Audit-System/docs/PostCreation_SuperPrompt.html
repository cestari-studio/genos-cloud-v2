Análise do Sistema Atual

O que existe hoje

  • Factory.tsx — Página principal com botão "Novo Post" que abre um Modal Carbon simples
  • Modal atual tem: Select (formato), TextInput (título), TextArea (descrição), DatePicker, hashtags, CTA, CardDataEditor, AI Instructions, CostEstimator, botão "Gerar com IA"
  • api.edgeFn('content-factory-ai', { action: 'generate' }) — gera conteúdo e retorna description, hashtags, cta, card_data
  • CostEstimator — chama token-estimator edge function para estimar custo
  • useCanGenerate — hook que verifica tokens/posts restantes
  • AuthContext — me.usage (tokens_used, tokens_limit, posts_used, posts_limit), me.wallet, me.config
  • PageLayout — Header com tenant name, badges AI inline (Tokens, Posts, Schedule), AILabel
  • MatrixList — DataTable com posts, status, formato, ações; refresh via ref
  • supabase.from('posts').insert() — insere post diretamente, sem geração AI obrigatória

O que precisa mudar

  • Substituir o Modal simples por um Modal Expressivo de IA com ProgressIndicator de 4 steps
  • Step 1: Tema principal (obrigatório)
  • Step 2: Formato + configuração específica (cards para carrossel, duração para reel, etc.)
  • Step 3: Detalhes adicionais (instruções AI, data, hashtags extras) + estimativa de custo
  • Step 4: Confirmação de custo + aprovação para gerar
  • Fase de Geração: Modal permanece aberto com ProgressBar mostrando etapas da IA
  • Resultado Final: Título gerado, descrição, preview do Audit Compliance Brand DNA, saldo utilizado, botão fechar
  • Post é inserido no banco automaticamente após geração
  • DataTable atualiza em tempo real
  • Wallet/badges/credits atualizam refletindo saldo consumido

Integração com Media Management System

  • O post criado pelo AI Flow terá status 'draft' — mídias são vinculadas posteriormente via MediaUploadModal
  • card_data gerado pela IA define as positions para assignment de mídias (P3/P4 do Media Management)
  • O format escolhido no Step 2 determina os limites de mídia (feed:1, carrossel:até 10, reels:1 vídeo only)
  • Após fechar o modal de resultado, o operador pode clicar no post na DataTable e fazer upload de mídias

P0 — Edge Function: post-creator (Refatorar content-factory-ai)

REGRA: Edge Function Deno/TypeScript em Supabase. Usa o client Supabase existente + OpenAI/Gemini.

TAREFA: Refatorar a action 'generate' do content-factory-ai para suportar o novo fluxo.

CONTEXTO EXISTENTE:
- api.edgeFn('content-factory-ai', { action: 'generate', tenantId, topic, targetFormat })
- Retorna: { description, hashtags, cta, ai_instructions, card_data }
- api.edgeFn('token-estimator', { tenantId, format, operation, slide_count, ai_model })
- Retorna: { estimated_cost, current_balance, after_balance, can_afford, breakdown }

MODIFICAÇÕES NA action 'generate':

Input ATUAL:
{ action: 'generate', tenantId, topic, targetFormat }

Input NOVO:
{
  action: 'generate',
  tenantId: string,
  topic: string,               // Step 1: tema principal
  context: string,             // Step 1: contexto adicional
  targetFormat: 'feed' | 'carrossel' | 'stories' | 'reels',  // Step 2
  cardCount: number,           // Step 2: qtd cards (carrossel) ou frames (stories)
  reelDuration: number | null, // Step 2: duração reels (15/30/60/90)
  feedMediaType: 'image' | 'video' | null, // Step 2: feed tipo mídia
  aiInstructions: string,      // Step 3: instruções extras
  scheduledDate: string | null,// Step 3: data agendada
  extraHashtags: string,       // Step 3: hashtags adicionais
  customCta: string,           // Step 3: CTA personalizado
}

FLUXO DA EDGE FUNCTION:

1. VALIDAR — Verificar permissão (JWT → tenant_members) + saldo de tokens
2. BUSCAR BRAND DNA:
   - SELECT brand_dna.* FROM brand_dna WHERE tenant_id = tenantId
   - Extrair: tone_of_voice, content_rules, hashtag_strategy, editorial_pillars, char_limits
   - fixed_description_footer de content_rules
   - fixed_hashtags de hashtag_strategy
   - char_limits por trecho (description, carousel_title, carousel_card, etc.)

3. MONTAR SYSTEM PROMPT para LLM:
   ```
   Você é o Helian v1.0, assistente de criação de conteúdo da genOS.
   
   BRAND DNA DO CLIENTE:
   - Tom de voz: ${tone_of_voice}
   - Regras de conteúdo: ${JSON.stringify(content_rules)}
   - Pilares editoriais: ${JSON.stringify(editorial_pillars)}
   
   LIMITES DE CARACTERES:
   - Descrição: máx ${char_limits.description || 300} chars
   - Título do carrossel: máx ${char_limits.carousel_title || 60} chars
   - Texto do card: máx ${char_limits.carousel_card || 150} chars
   - Título de reel: máx ${char_limits.reel_title || 40} chars
   - Título estático: máx ${char_limits.static_title || 60} chars
   - Parágrafo estático: máx ${char_limits.static_paragraph || 200} chars
   
   HASHTAGS:
   - Fixas (OBRIGATÓRIO incluir): ${JSON.stringify(fixed_hashtags)}
   - Máximo total: ${hashtag_strategy.max_total || 5}
   - Gerar restantes: ${hashtag_strategy.generate_remaining ? 'Sim' : 'Não'}
   
   ASSINATURA FIXA:
   Ao final de TODA descrição, SEMPRE incluir:
   ${fixed_description_footer || '(nenhuma)'}
   
   FORMATO DO POST: ${targetFormat}
   ${targetFormat === 'carrossel' ? `Gerar ${cardCount} cards com título e texto para cada.` : ''}
   ${targetFormat === 'reels' ? `Duração estimada: ${reelDuration}s. Gerar script/roteiro.` : ''}
   ${targetFormat === 'stories' ? `Gerar ${cardCount} frames com texto para cada.` : ''}
   
   INSTRUÇÕES EXTRAS DO OPERADOR:
   ${aiInstructions || '(nenhuma)'}
   
   HASHTAGS EXTRAS DO OPERADOR:
   ${extraHashtags || '(nenhuma)'}
   
   CTA PERSONALIZADO:
   ${customCta || '(gerar um CTA relevante)'}
   ```

4. GERAR CONTEÚDO (chamada à LLM):
   Response esperada em JSON:
   {
     "title": "string",
     "description": "string (incluindo footer fixo ao final)",
     "hashtags": "string (fixas + geradas)",
     "cta": "string",
     "card_data": [
       { "position": 1, "cardTitulo": "...", "cardTexto": "...", "text_primary": "...", "text_secondary": "..." },
       ...
     ],
     "ai_instructions": "string (resumo do que foi gerado)",
     "audit": {
       "brand_voice_score": number (0-100),
       "char_limits_ok": boolean,
       "fixed_hashtags_included": boolean,
       "fixed_footer_included": boolean,
       "overall_score": number (0-100),
       "notes": ["string"]
     }
   }

5. INSERIR POST NO BANCO:
   INSERT INTO posts (
     tenant_id, format, status, title, description,
     scheduled_date, hashtags, cta, card_data,
     media_slots, ai_instructions, ai_model, ai_audit
   ) VALUES (...)
   status = 'draft'
   
   → Retornar o post completo incluindo id

6. DEBITAR TOKENS:
   - Calcular custo real (baseado em usage da API)
   - UPDATE usage SET tokens_used = tokens_used + custo_real WHERE tenant_id = tenantId
   - INSERT INTO token_transactions (tenant_id, amount, operation, post_id, ...)

7. RETORNAR:
   {
     success: true,
     post: { id, title, description, hashtags, cta, card_data, format, status, scheduled_date },
     audit: { brand_voice_score, char_limits_ok, fixed_hashtags_included, fixed_footer_included, overall_score, notes },
     cost: { tokens_consumed, new_balance, posts_used, posts_limit }
   }
