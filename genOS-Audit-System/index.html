🔴 Vulnerabilidades Identificadas

● C1 Endpoint Não Protegido

O wix-auth-bridge tem verify_jwt: false. Qualquer pessoa pode chamar o endpoint e criar tenants.

Fix: Adicionar header secreto compartilhado (ex: X-Bridge-Secret) ou habilitar JWT e usar token de serviço.

● C2 Senha Determinística

Senha = genOS!{tenant_id[0:8]}{serviceRoleKey[-6:]}. Se a service role key vazar, TODAS as senhas são comprometidas.

Fix: Gerar senha aleatória com crypto.randomUUID() e NÃO armazenar (usar apenas no momento da criação).

● C3 View Expõe auth.users

A view v_user_tenant_roles é SECURITY DEFINER e expõe dados de auth.users para a role anon.

Fix: Converter para SECURITY INVOKER ou restringir acesso via RLS.

● C4 User ID = Tenant ID

O ID do usuário no auth.users é o mesmo que o tenant_id. Permite enumeração e colisões.

Fix: Usar UUID aleatório para auth.users e manter tenant_id nos metadata.

🔴 Bug Fix — Social Connect Buttons

3 bugs encontrados e documentados nos botões de conexão Instagram/Facebook

Bugs
3
1 Critical + 2 Medium
Root Cause
100% Frontend
Nenhuma alteração na edge fn
Arquivos
2
SocialConnectionsTab + SocialCallback
🐛 Bug 1 — Double Destructure no handleConnect
● CRITICAL — Botão não faz nada

Sintoma: Clicar "Conectar Instagram/Facebook" não abre nenhuma janela

Causa: api.edgeFn() já faz unwrap de {data:...}. O componente faz segundo destructure → undefined

Arquivo: SocialConnectionsTab.tsx (linhas 48-59)

Fix: const result = await api.edgeFn(...); em vez de const { data } = await api.edgeFn(...);

🐛 Bug 2 — Parâmetros errados no handleDisconnect
● MEDIUM — Desconectar retorna erro 500

Sintoma: Clicar "Desconectar" retorna erro 500

Causa: Frontend envia connectionId (camelCase) mas edge function espera connection_id (snake_case) + falta tenant_id

Arquivo: SocialConnectionsTab.tsx (linhas 61-69)

Fix: Enviar connection_id + tenant_id: activeTenantId

🐛 Bug 3 — Falta ig_account_id no select_page
● MEDIUM — Instagram não conecta via Facebook Login

Sintoma: Após autenticar e selecionar página, conta IG Business NÃO é salva

Causa: Callback não envia ig_account_id nem provider para a edge function

Arquivo: SocialCallback.tsx (linhas 69-76)

Fix: Adicionar ig_account_id: page.instagram_business_account?.id || null e provider: stateData.provider

📋 Padrão Importante

Regra: api.edgeFn() já faz unwrap de {data: ...}. NUNCA fazer const { data } = await api.edgeFn(...). Sempre usar const result = await api.edgeFn(...)
