  • DatePicker: ✅ JÁ EXISTE no Factory.tsx (linhas 246-264) E no preview modal do MatrixList (linhas 1050-1067). Mas falta opção "IA escolhe data" (hoje, amanhã, essa semana, etc).

FIX-2 — Smart Refresh (sem polling agressivo)

O AuthContext faz polling a cada 60s que causa re-render em cascata desnecessário.

## FIX-2 — SMART REFRESH (SEM POLLING AGRESSIVO)

### REGRA CARBON ABSOLUTA
100% @carbon/react. ZERO Tailwind. ZERO inline styles hardcoded.

### CONTEXTO
O AuthContext (linha 67) faz `setInterval(() => refreshMe(undefined, true), 60_000)`.
Isso causa re-render GLOBAL a cada 60s porque atualiza o state `me` inteiro.
O resultado: todas as páginas "piscam" mesmo sem mudança real nos dados.

### ALTERAÇÃO

#### AuthContext.tsx — Polling inteligente com comparação

Substituir o useEffect (linhas 65-69) por:

```typescript
useEffect(() => {
  refreshMe(); // Initial load

  const interval = setInterval(async () => {
    try {
      const fresh = await api.getMe(false); // NÃO forçar cache bypass no background

      // Proteger contra falha transitória
      if (meRef.current.authenticated && !fresh.authenticated) return;

      // SÓ atualizar se dados REALMENTE mudaram
      const currentHash = `${meRef.current.wallet?.credits}:${meRef.current.usage?.tokens_used}:${meRef.current.usage?.posts_used}`;
      const freshHash = `${fresh.wallet?.credits}:${fresh.usage?.tokens_used}:${fresh.usage?.posts_used}`;

      if (currentHash !== freshHash) {
        setMe(fresh);
      }
      // Se hash igual, NÃO chamar setMe → zero re-render
    } catch (err) {
      console.warn('genOS AuthContext: background poll error — keeping current state');
    }
  }, 120_000); // ← 120s em vez de 60s (o refresh explícito cuida do resto)

  return () => clearInterval(interval);
}, []);
```

### POR QUE 120s?
- O polling agora é APENAS safety net para detectar mudanças feitas fora da tab (ex: outro admin alterou config)
- Todas as operações de token (gerar, regenerar, aprovar) já fazem `refreshWallet()` explícito
- Reduz chamadas API pela metade sem impacto percebido

### VALIDAÇÃO
1. Abrir Content Factory → NÃO deve piscar a cada 60s
2. Gerar post → saldo atualiza IMEDIATAMENTE (via refreshWallet explícito)
3. Deixar tab aberta 5min → se nenhuma operação foi feita, zero re-renders (verificar via React DevTools Profiler)
