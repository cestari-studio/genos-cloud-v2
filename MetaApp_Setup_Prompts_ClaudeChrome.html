Ou no Graph API Explorer, use o token atual **Processo:** 1. Com o User Token atual (curta duração ~1h), execute: GET: `/oauth/access_token?grant_type=fb_exchange_token&client_id=852899784442624&client_secret={APP_SECRET}&fb_exchange_token={SHORT_LIVED_TOKEN}` 2. O resultado será um Long-Lived User Token (~60 dias) 3. Com o Long-Lived User Token, busque o Page Token que já é long-lived: GET: `/me/accounts?fields=id,name,access_token` O `access_token` retornado para cada Page JÁ É long-lived (e não expira se o usuário é admin permanente da Page) **ATENÇÃO:** - O Long-Lived User Token dura ~60 dias - O Page Access Token obtido via Long-Lived User Token NÃO expira (é permanente) - No genOS, o `meta-oauth` edge function faz essa troca automaticamente - Os tokens são armazenados no `vault.secrets` do Supabase com encriptação Anote os tokens para configurar no ambiente de teste.
    ✓ After completing this step
    • You should have a Long-Lived User Token (expires in ~60 days)
    • You should have Long-Lived Page Access Tokens (do not expire)
    • These tokens should be stored securely for later use
